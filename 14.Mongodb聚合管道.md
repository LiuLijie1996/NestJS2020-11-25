## MongoDB  聚合管道

```
使用聚合管道可以对集合中的文档进行变换和组合，返回新的数据。
实际项目：表关联查询、数据的统计。
MongoDB 中使用：
	db.表的名称.aggregate([{<stage>},{<stage>},...]) 方法 来构建和使用聚合管道。
```



## 管道操作符与表达式

```
$project   增加、删除、重命名字段。
$match   条件匹配。只满足条件的文档才能进入下一阶段。
$limit    限制结果的数量。
$skip   跳过文档的数量。
$sort   条件排序。
$group   条件组合结果 统计。
$lookup   用以引入其它集合的数据（表关联查询）。
```

### 管道表达式：

```js
管道操作符作为“键”,所对应的“值”叫做管道表达式。
例如：
    db.order.aggregate([
        {
            $match: {uid: 9}
        }
    ])
$match 称为管道操作符，而 {uid: 9} 称为管道表达式， 是管道操作符的操作数
每个管道表达式是一个文档结构，它是由字段名、字段值、和一些表达式操作符组成的。
```

### 常用表达式操作符

```
$addToSet   将文档指定字段的值去重
$max   文档指定字段的最大值
$min   文档指定字段的最小值
$sum   文档指定字段求和
$avg   文档指定字段求平均
$gt   大于给定值
$gte   大于等于给定值
$eq   等于给定值
$lt   小于给定值
```



## $project

```ts
### 修改文档的结构，可以用来重命名、增加或删除文档中的字段。

// 要求查找 order 只返回文档中 trade_no 和 all_price 字段
db.order.aggregate([
    {
        $project: {
            trade_no: 1,
            all_price: 1,
        }
    }
])

/*
    所得结果：
        { "_id" : ObjectId("5fcc9375979e8a673eb1536d"), "trade_no" : "111", "all_price" : 100 }
        { "_id" : ObjectId("5fcc93b9979e8a673eb1536e"), "trade_no" : "222", "all_price" : 90 }
        { "_id" : ObjectId("5fcc93bd979e8a673eb1536f"), "trade_no" : "333", "all_price" : 20 }
*/
```



## $match

```ts
### 作用：用于过滤文档。用法类似于 find() 方法中的参数。

// 要求查找 order 只返回文档中 all_price 大于 90 的数据
db.order.aggregate([
    {
        $project: {
            trade_no: 1,
            all_price: 1,
        }
    },
    {
        $match: {
            all_price: {$gte: 90}
        }
    },
])

/*
	所得结果：
        { "_id" : ObjectId("5fcc9375979e8a673eb1536d"), "trade_no" : "111", "all_price" : 100 }
        { "_id" : ObjectId("5fcc93b9979e8a673eb1536e"), "trade_no" : "222", "all_price" : 90 }
*/
```



## $group

```ts
### 将集合中的文档进行分组，可用于统计结果。 

// 统计每个订单的订单数量，按照订单号分组
db.order_item.aggregate([
    {
        $group: {
            _id: "$order_id", // 将 order_id 字段相同的订单融合成一个对象
            total: {$sum: "$num"}, // 将 num 字段进行求和,求和的结果放在total中
        }
    },
])

/*
	所得结果：
        { "_id" : "1", "total" : 3 }
        { "_id" : "2", "total" : 2 }
        { "_id" : "3", "total" : 6 }
*/
```



## $sort

```ts
### 将集合中的文档进行排序。

db.order.aggregate([
	{//限制显示的字段
        $project:{trade_no: 1, all_price: 1}
    },
	{//过滤
        $match:{"all_price": {$gte: 90}} //大于等于90
    },
	{//排序
        $sort: {"all_price": -1},//-1从高到低，1从低到高
    }
])

/*
	所得结果：
		{ "_id" : ObjectId("5fcc9375979e8a673eb1536d"), "trade_no" : "111", "all_price" : 100 }
		{ "_id" : ObjectId("5fcc93b9979e8a673eb1536e"), "trade_no" : "222", "all_price" : 90 }
*/
```



## $limit

```ts
### 限制结果的数量

db.order.aggregate([
	{//限制显示的字段
        $project:{trade_no: 1, all_price: 1}
    },
    {//限制显示的数量
        $limit: 1,
    }
])

/*
	所得结果：
		{ "_id" : ObjectId("5fcc9375979e8a673eb1536d"), "trade_no" : "111", "all_price" : 100 }
*/
```



## $skip

```ts
### 跳过文档的数量

db.order.aggregate([
	{//限制显示的字段
        $project:{trade_no: 1, all_price: 1}
    },
    {//跳过文档的数量
        $skip: 1,
    },
    {//限制显示的数量
        $limit: 1,
    },
])

/*
	所得结果：
		{ "_id" : ObjectId("5fcc93b9979e8a673eb1536e"), "trade_no" : "222", "all_price" : 90 }
*/
```
